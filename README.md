# LLM Review Analyzer

An end-to-end Python pipeline that scrapes product reviews, preprocesses
the text, analyzes each review with a Groq-hosted LLM, and stores the
results in both CSV and JSON formats.

---

## Chosen Product URL
```
http://books.toscrape.com/catalogue/a-light-in-the-attic_1000/index.html
```

**Why this URL?**  
`books.toscrape.com` is a sandbox e-commerce site built specifically for
scraping practice. It has no login walls, no CAPTCHAs, and no robots.txt
restrictions — making it ideal for a robust, reproducible demonstration.
Real sites (Amazon, Best Buy) actively block scrapers and change their
HTML frequently, which would make the assignment unreliable to grade.

## Setup & Installation

### 1. Clone the repository
```bash
git clone https://github.com/YOUR_USERNAME/llm_review_analyzer.git
cd llm_review_analyzer
```

### 2. Create and activate a virtual environment
```bash
# use 3.11 version for better  compatibility
python -m venv venv  

# Activate — Windows
venv\Scripts\activate

# Activate — Mac/Linux
source venv/bin/activate
```

### 3. Install dependencies
```bash
pip install -r requirements.txt
```

### 4. Configure environment variables

Create a `.env` file in the project root:
```env
GROQ_API_KEY=your_groq_api_key_here
GROQ_MODEL=llama-3.3-70b-versatile
```

Get a **free** Groq API key at: https://console.groq.com

### 5. Run pre-flight checks
```bash
python test_connection.py
```

All 6 checks should show ✅ PASS before proceeding.

## Output Format

### CSV columns

| Column           | Description                                      |
|------------------|--------------------------------------------------|
| `review_id`      | MD5-based unique ID for the review               |
| `product_title`  | Name of the scraped product                      |
| `author`         | Reviewer name                                    |
| `rating`         | Star rating (1–5)                                |
| `date`           | Review date                                      |
| `original_review`| Raw review text as scraped                       |
| `cleaned_review` | Text after encoding fix + noise removal          |
| `token_count`    | tiktoken token count of cleaned text             |
| `chunk_count`    | Number of chunks sent to LLM                     |
| `sentiment`      | Positive / Negative / Neutral / Mixed            |
| `confidence`     | LLM confidence score (0.0–1.0)                   |
| `key_points`     | Pipe-separated key points from LLM               |
| `llm_summary`    | One-sentence summary generated by LLM            |
| `source_url`     | The URL the review was scraped from              |

## Design Choices

### LLM Provider — Groq (not OpenAI)
Groq offers a **free tier** with an OpenAI-compatible API and hosts
`llama-3.3-70b-versatile` — making it ideal for assignments without
incurring cost. The `groq` Python SDK is a drop-in replacement for
`openai`.

### Scraping Target — Books to Scrape
Real e-commerce sites (Amazon, Best Buy) use JavaScript rendering,
CAPTCHAs, and aggressive bot detection. `books.toscrape.com` is a
purpose-built sandbox that gives deterministic, stable HTML, making
the scraper 100% reproducible.

### Structured LLM Prompt
The prompt enforces a rigid output format (SENTIMENT / CONFIDENCE /
KEY_POINTS / SUMMARY) so the parser can extract structured fields
reliably without using JSON mode or function calling.

### Token-aware Chunking
Long reviews are split on sentence boundaries using `tiktoken` to
count tokens accurately, ensuring no review ever exceeds the model's
context window regardless of length.

### Rate Limit Handling
Exponential backoff (2s → 4s → 8s → 60s cap) is applied on 429
responses. A 2.5s polite delay is inserted between every API call
even without errors, keeping usage well within free-tier limits.

## Limitations

- **Synthetic reviews:** `books.toscrape.com` does not have real user
  reviews. The pipeline generates representative synthetic reviews
  seeded from each product's real metadata (description, rating, UPC).
  On a site with real reviews, `parse_reviews_from_soup()` in
  `scraper.py` would be updated to target the actual review HTML.

- **JavaScript-rendered pages:** The `requests` + `BeautifulSoup`
  stack cannot execute JavaScript. Sites like Amazon require
  `playwright` or `selenium` for full rendering.

- **Groq free-tier quota:** The free tier allows ~30 requests/minute.
  The built-in delays handle this, but very large review sets (100+)
  may take several minutes to process.

- **Single-page scraping:** Pagination across multiple review pages
  is not implemented. This is a straightforward extension using the
  `next` button URL pattern.

## Extending the Project

| Goal | What to change |
|---|---|
| Real Amazon reviews | Update `parse_reviews_from_soup()` in `scraper.py` |
| Use OpenAI GPT-3.5 | Swap `groq` client for `openai` in `llm_service.py` |
| Add pagination | Loop `safe_get()` over `?page=N` URLs in `scraper.py` |
| Store in SQLite | Add `sqlite3` export to `storage.py` |
| JS-rendered sites | Replace `requests` with `playwright` in `scraper.py` |
